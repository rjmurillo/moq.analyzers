# Moq1600: Protected setup should use `ItExpr` matchers

| Item     | Value   |
| -------- | ------- |
| Enabled  | True    |
| Severity | Warning |
| CodeFix  | False   |

---

When setting up or verifying protected members using the string-based overloads of `mock.Protected().Setup(...)` or
`mock.Protected().Verify(...)`, you must use `ItExpr` matchers instead of `It` matchers. Using `It` matchers compiles
but fails at runtime because the protected API uses `Expression`-based matching, not delegate-based matching.

The lambda-based overload via `.As<TInterface>()` correctly uses `It` matchers, so this rule only applies to
string-based overloads.

## Examples of patterns that are flagged by this analyzer

```csharp
using Moq;
using Moq.Protected;

public abstract class MyBase
{
    protected virtual bool Foo(string arg) => false;
}

var mock = new Mock<MyBase>();

// Moq1600: uses It.IsAny instead of ItExpr.IsAny
mock.Protected()
    .Setup<bool>("Foo", It.IsAny<string>())
    .Returns(true);
```

## Solution

```csharp
using Moq;
using Moq.Protected;

public abstract class MyBase
{
    protected virtual bool Foo(string arg) => false;
}

var mock = new Mock<MyBase>();

// Correct: uses ItExpr.IsAny
mock.Protected()
    .Setup<bool>("Foo", ItExpr.IsAny<string>())
    .Returns(true);
```

## When this rule does not apply

The lambda-based protected API uses standard `It` matchers. This is correct and will not trigger the rule:

```csharp
public interface IMyProtectedMembers
{
    bool Foo(string arg);
}

mock.Protected()
    .As<IMyProtectedMembers>()
    .Setup(m => m.Foo(It.IsAny<string>()))
    .Returns(true);
```

## Suppress a warning

If you just want to suppress a single violation, add preprocessor directives to
your source file to disable and then re-enable the rule.

```csharp
#pragma warning disable Moq1600
mock.Protected()
    .Setup<bool>("Foo", It.IsAny<string>())
    .Returns(true);
#pragma warning restore Moq1600
```

To disable the rule for a file, folder, or project, set its severity to `none`
in the
[configuration file](https://learn.microsoft.com/en-us/dotnet/fundamentals/code-analysis/configuration-files).

```ini
[*.{cs,vb}]
dotnet_diagnostic.Moq1600.severity = none
```

For more information, see
[How to suppress code analysis warnings](https://learn.microsoft.com/en-us/dotnet/fundamentals/code-analysis/suppress-warnings).
